Reset

Команда git reset имеет 3 режима работы

  1. git reset --soft HEAD~1мы откатим коммит но все изменения останутся staged "зеленый файл в консоли" HEAD~1 - указывает что откатится на один относительно текшего положения head
  2. git reset HEAD~1мы откатим коммит но все изменения останутся unstaged "красный файл в консоли"
  3. git reset --hard HEAD~1мы откатим коммит и удаляем все изменения без возможности восстановления
Удаление веток

При удаление ветки мы не должны находится в удаляемой ветке

При удаление ветки все коммиты в этой ветке тоже будут удалены для сохранения изменений в ветки ее сначала нужно слить с другой веткой а затем удалить

  • git branch -d branch-name первый способ удаления ветки
  • git branch --delete branch-nameвторой способ удаления ветки
detached commit
  1. С помощью команды git checkout №коммита переходим к нужному коммиту для висения изменений он будет detached
  2. Вносим изменения
  3. git add .
  4. git commit -m "Внесли изменения"
  5. git switch master после переключения мы теряем наш коммит но у нас есть его id
  6. git branch branch-name id-commit для того чтобы не потерять коммит мы создаем ветку и вновь созданную ветку добавляем наш коммит
  7. git merge branch-name сливаем вновь созданную ветку с веткой мастер для сохранения изменений

решение конфликта при вливание

git log --mergeпоказывает какие коммиты конфликтуют

В vs code моно решить конфликт варианты решения ниже

  1. отредактировать в ручную
  2. Accept current changeПринять текущие изменение - это то которое было в главной ветке до вливания
  3. Accept incoming changeПринять входящие изменение - это то которое вливаем
  4. Accept both changesПринять оба изменения
  5. Compare changesсравнить изменения
