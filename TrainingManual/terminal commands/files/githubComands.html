Команды гит

Пошагово старт работы с гит на компьютере

  1. Создать папку на компе и открыть из нее консоль или открыть консоль в другом месте в консоли написать команду cd и перетащить папку в консоль чтобы прописать путь автоматом
  2. ls – проверка cодержимого папки – выводит в консоль (папка должна быть пустой)
  3. Генерим ssh ключ:
    1. вводим в консоль команду ssh-keygen
    2. затем программа просит ввести пароль можно не вводить просто нажать enter
    3. Затем подтверждение пароля
    4. И в папке c/users/user/.ssh добавится файл id-rsa.pab открываем его в блокноте копируем ключ (ctr+a, ctr+c)
    5. И в настройках на сайте гит-хаб вводим этот ключ
  4. (!Срабатывает со второго раза, сначала пишет что нужно ввести просто clone) Вводим в консоль находясь в папке проекта “git clone + адрес с сайте git hub(который мы получаем из: заходим в нужный нам репозиторий нажимаем зеленую кнопку code в открытом окне выбираем ssh и там чуть ниже появится ссылка для копирования которую нужно вставить после git clone)” Справка «git--help» – команды гита
  5. Открываем папку которая с качалась с git в редакторе кода и также в консоли тоже меняем путь до папки
  6. Команды Git:
    1. git status => показывает статус (все изменённые файлы светятся красным)
    2. git add . git add . => добавляет файлы в гит (в статусе файл светится зеленом)
    3. git commit - - m “сообщение” => делает коммит
    4. git push git push => отправляет изменения на удаленный репозиторий
    5. работа с ветками
    6. git checkout – b name branch => создает ветку с названием name branch
    7. git branch => показывает ветку в которой я нахожусь
    8. git checkout master => переключились на мастер
    9. когда пушим новую ветку там гитхаб подскажет что нужно дописать к команде git push и на удаленном репозитории появится новая ветка
    10. для того чтобы слить новую ветку с веткой мастер нужно нажать зеленую кнопку
      • compare pull request
      • create pull request
      • marge pull request
      • если изменение не конфликтует confirm marge
      • git pull – для того чтобы внести изменения в локальный репозиторий
    11. git branch -- delete name branch – удалить ветку name branch

    git reset HEAD~1 – откат камита в рабочею директорию где цифра количество кaмитов

Минин

        
            Основные команды
git init
Позволяет проинициализировать репозиторий в текущей папке

git status
Показывает текущий статус

git add
Отслеживает изменения файлов
git add index.html — добавляет index.html
git add . — добавляет все файлы

git commit
Сохраняет изменения в коммит
git commit -m 'commit message' — создает коммит с сообщением

git branch
Работа с ветками в репозитории
git branch — показывает список веток
git branch branch-name — создает новую ветку branch-name
git branch -D branch-name — удаляет ветку branch-name

git checkout
Переключается на другую ветку
git checkout branch-name — переключается на последний коммит в ветке
branch-name
git checkout -b branch-name — создает и переключается на ветку branch-name

git merge
Совмещает текущую ветку с выбранной
git merge branch-name — совмещает текущую ветку с branch-name

git config
Конфигурация и параметры git
git config --global user.name — Показывает имя пользователя
git config --global user.name 'new user' — Изменяет имя пользователя
git config --global user.email — Показывает email пользователя
git config --global user.email 'test@mail.ru' — Изменяет email пользователя

git push
Заливает текущие локальные коммиты в удаленный репозиторий

git pull
Забирает изменения с удаленного репозитория в локальный

git clone
Клонирует проект с удаленного репозитория
